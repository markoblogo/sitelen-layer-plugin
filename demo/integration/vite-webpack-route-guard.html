v0.3 demo: route-aware build plugins

Use this page as a reference for CI/static docs snapshots and smoke runs.

Vite plugin route config

import sitelenLayerVitePlugin from 'sitelen-layer-plugin/vite-plugin'; export default defineConfig({ plugins: [ sitelenLayerVitePlugin({ autoImportStyle: true, autoInit: true, routeInclude: ['/tp', '/toki-pona'], routeExclude: ['/admin', '/api/'], routeMatchMode: 'contains' }) ] });

Webpack plugin route config

const { SitelenLayerWebpackPlugin } = require('sitelen-layer-plugin/webpack-plugin'); new SitelenLayerWebpackPlugin({ autoInjectStyles: true, autoInit: true, routeInclude: ['/tp', '/toki-pona'], routeExclude: ['/admin', '/login'], routeMatchMode: 'contains' });

Smoke checks

  • allowed routes should include plugin init script
  • blocked routes should skip plugin init script
  • rebuild output must contain only one injected style/script marker
