## B) Build-плагины с route guard (allowlist/denylist)

`vite`/`webpack` helper-плагины теперь поддерживают маршруты:

```ts
sitelenLayerVitePlugin({
  autoInit: true,
  routeInclude: ['/tp', '/toki-pona'],
  routeExclude: ['/admin', '/api/'],
  routeMatchMode: 'contains'
});
```

```js
new SitelenLayerWebpackPlugin({
  autoInit: true,
  routeInclude: ['/tp', '/toki-pona'],
  routeExclude: ['/admin', '/login'],
  routeMatchMode: 'contains'
});
```

Режимы сопоставления: `contains | startsWith | exact | regex`.

## C) Ключевые требования для v0.3.x

- Единый модульный wrapper/пакет для каждой CMS вместо копипаста.
- Конфигурируемый route allow/deny для безопасного автозапуска.
- Разделение `build`/`runtime`/`storage`-потока и стабильный pipeline обновления маппингов.

## D) Что считать завершённым в минимальном PoC

- [ ] Browser extension: route-guard в content/background + ручной toggle.
- [ ] Obsidian/Logseq: корректный init/destroy lifecycle и re-hydrate на route change.
- [ ] WP/Ghost: безопасный шаблон инъекции и базовая проверка CSP/маршрутов.
- [ ] Дальнейший шаг: разнести PoC в отдельные пакеты/репозитории с версионным релизом.
