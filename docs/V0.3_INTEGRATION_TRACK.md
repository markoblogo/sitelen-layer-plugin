## B) Build-плагины с route guard (allowlist/denylist)

`vite`/`webpack` helper-плагины поддерживают маршруты:

```ts
sitelenLayerVitePlugin({
  autoInit: true,
  routeInclude: ['/tp', '/toki-pona'],
  routeExclude: ['/admin', '/api/'],
  routeMatchMode: 'contains'
});
```

```js
new SitelenLayerWebpackPlugin({
  autoInit: true,
  routeInclude: ['/tp', '/toki-pona'],
  routeExclude: ['/admin', '/login'],
  routeMatchMode: 'contains'
});
```

Режимы сопоставления: `contains | startsWith | exact | regex`.

## D) Что считать завершённым в минимальном PoC

- [x] Browser extension: route-guard в `content.js` + guard на двойной init.
- [x] Obsidian/Logseq: корректный `init`/`destroy`/re-init lifecycle.
- [x] WP/Ghost: безопасный шаблон инъекции с route-фильтрами.

## F) Публикация v0.3.0-MVP (preflight pipeline)

- CI выполняет `npm run build`, `npm run pack:adapters`, `npm run smoke:adapters`, `npm run smoke:publication`.
- Для каждого адаптера создаётся zip-артефакт в `dist/adapters/<pkg>-<adapter>-v<version>.zip`.
- В workflow архивируются готовые адаптер-пакеты и npm-транспортный `.tgz` для ручной проверки.
